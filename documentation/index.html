About the API

Plato is an application created for study management. Both developers of the app are former teachers that believed students could use a better way to organize and prepare for their studies. In order to do that, we thought of bringing the most efficient tools of project management and adapt them for students.
The Plato API was created to manage data received, updated and deleted on the application.
On this document you will find details on how to use each route of the API and get the best out of it.

Authentication

Plato API currently has both private and public routes, although they are mostly private
You will not need any authentication to access our public routes.
To access our private routes, you will need to have an Access Token, that is only provided to signed and logged users.
Signing up and Logging in are both public routes. Once you log in, you will receive a token. Set it as your environmental token to access the private routes.
It is important to notice that, even after authenticated, users will only be able to check their on information on most routes (does not apply for the GET ALL BOOKS route)

Authentication Routes

These routes are intended to help users create an account and log into the platform. They are both public routes that do not demand a token to be used.

Type: Public Route
Authentication: Not Required

Method Endpoint Payload Response Action
POST /auth/signup { "email": string,
"password": string,
"name": string }		 { user } Creates a new user
POST /auth/login { "email": string,
"name": string }		 { token } Login user

User Routes

These routes are intended to help users manage and update their information, as well as updating their profile picture.

Type: Private Route
Authentication: Token Required

Method Endpoint Payload Response Action
GET /user - { user } Get logged user
PUT /user { "name": string,
"email": string }		 { user } Updated logged user
PUT /user/profile-image { "image": image } { user } Updated logged user image

Book Routes

These routes are intended to help users find, favorite or create books to add to their future readings.

Type: Private Route
Authentication: Token Required

Method Endpoint Payload Response Action
GET /book - [books] Get all books
GET /book/:bookId - { book } Updated logged user
POST /book { "name": string,
"author": string }		 { book } Creates a book
PUT /book/image/:bookId { "image": image } { book } Update a book cover
DELETE /book/:bookId - - Deletes a book

Tasks Routes

These routes are intended to help users create, organize and delete their tasks

Type: Private Route
Authentication: Token Required

Method Endpoint Payload Response Action
GET /task - [tasks] Get all tasks
POST /task { "title": string } { task } Creates a task
PUT /task/:taskId { "title": string } { task } Update a task
DELETE /task/deleteOne/:taskId - - Deletes a task
DELETE /task/deleteAll - - Deletes all tasks

Event Routes

These routes are intended to help users create, organize and delete their events

Type: Private Route
Authentication: Token Required

Method Endpoint Payload Response Action
GET /event - [events] Get all events
POST /event { "title": string,
"description": string,
"date": date }		 { event } Creates a event
PUT /event/:eventId { "title": string,
"description": string,
"date": date }		 { event } Update a event
DELETE /event/deleteOne/:eventId - - Deletes a event
DELETE /event/deleteAll - - Deletes all events
